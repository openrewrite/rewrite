# Phase 0 — file-level specification

2026-07-07. Companion to `DESIGN.md` §7–8. Phase 0 builds the machinery that makes every later
phase safe: the shape-aware snapshot/diff, the determinism and identity gates, corpus +
record/replay + ground-truth tooling, and the ledger. No production resolution behavior changes
in this phase; the only production-adjacent addition is the (inert until Phase 2) engine
selector.

## 1. Engine selection (dev/CI-only, defined now, wired in Phase 2)

- Internal message key `org.openrewrite.maven.resolution.engine` ∈ `legacy` (default) | `maven`
  | `shadow`; default read from the same-named system property.
- Read via a new internal helper `org.openrewrite.maven.internal.ResolutionEngineSelector`
  (package `internal` — deliberately NOT a `MavenExecutionContextView` method; per the
  no-user-facing-toggle decision this never becomes public API).
- CI shadow job = `-Dorg.openrewrite.maven.resolution.engine=shadow ./gradlew :rewrite-maven:test`.
  No JUnit extension: the comparison lives inside the facades (`Pom.resolve`,
  `ResolvedPom.resolveDependencies`), so every resolution — tests, recipes, `UpdateMavenModel`,
  rewrite-gradle headless — is compared, not just ones a test hook can see.
- Shadow semantics (implemented Phase 2): run legacy, run engine, snapshot both, diff; non-empty
  diff → `AssertionError` carrying the rendered diff, unless every diff path is masked by a
  ledger id (§5).

## 2. Snapshot and diff (slice A — in `rewrite-maven/src/test/java/org/openrewrite/maven/parity/`)

The one component where lossiness is fatal (nothing downstream catches an oracle that can't see
a difference). All classes are test-scope in Phase 0; they move to a `testFixtures` source set
in Phase 2 so rewrite-gradle tests can reuse them.

| File | Responsibility |
|---|---|
| `ResolutionSnapshot.java` | Canonical projection of a `MavenResolutionResult` (+ thrown errors + listener events) into a deterministic, JSON-serializable form. Static factory `ResolutionSnapshot.of(MavenResolutionResult, List<Throwable>, List<Event>)`. |
| `ResolutionDiff.java` | Structural diff of two snapshots → ordered list of `(jsonPath, left, right)`; `isEmpty()`; `render()` for assertion messages. |
| `SnapshotNormalizer.java` | The normalization rules (below) + the known-nondeterminism mask registry. |
| `RecordingResolutionListener.java` | `ResolutionEventListener` capturing events as a multiset of `type:key` strings. |
| `DeterminismTest.java` | Old-vs-old gate: resolve each fixture twice with fresh, isolated in-memory caches; assert empty diff. Doubles as a nondeterminism audit of the legacy engine — findings become ledger entries + masks, not silent tolerance. |
| `IdentityContractsTest.java` | Pins the `==` contracts: `getResolvedDependency(each requested) != null`; `getResolvedManagedDependency` for `Defined`/`Imported`; `resolve()` no-change returns same instance; within-scope instance sharing between flat list and nested graph. Engine-independent — survives the swap unchanged. |
| `SerializedLstCompatibilityTest.java` | Deserializes committed `MavenResolutionResult` payloads (`src/test/resources/parity/serialized-lsts/`) and asserts field-level expectations. Payloads: current-version capture now (pins the wire format forward) + payloads generated by an older released rewrite version (fetch via a small generator run against `org.openrewrite:rewrite-maven:<old>` on an isolated classpath — same source-set isolation trick as the benchmark spike). |
| `fixtures/` (test resources) | Curated pom fixture sets covering: parent chains, profiles (property/jdk activation), DM + single- and multi-level BOM import, conflicts at equal/unequal depth, exclusions, optional, classifiers, snapshots (pinned + dated), plugins with executions, property indirection. Each fixture = a directory of poms + a `repo/` of remote poms served via `file://` or MockWebServer. |

### Snapshot format (deterministic JSON, sorted keys except where order is semantic)

```json
{
  "pom": {
    "gav": "g:a:v", "packaging": "jar",
    "properties": {"k": "raw-value"},
    "dependencyManagement": [{"gact": "...", "version": "...", "scope": "...",
      "exclusions": ["g:a"], "bomGav": "g:a:v|null", "requestedRef": "dm[3]|bom[1]"}],
    "repositories": [{"id": "...", "uri": "..."}],
    "plugins": [{"ga": "...", "version": "...", "executions": ["..."]}]
  },
  "scopes": {
    "Compile": [{"i": 0, "gav": "g:a:v", "dated": null, "depth": 0, "repo": "uri",
      "requestedRef": "root[2]|node[4]", "children": [1, 3]}]
  },
  "errors": [{"type": "MavenDownloadingException", "message": "<normalized first line>"}],
  "events": {"downloadSuccess:g:a:v": 2},
  "identity": {"requestedThreading": true, "resolveNoChange": true, "dmThreading": true}
}
```

Deliberately captured (the critiques' loss list): per-scope **ordering**, nested graph shape via
`children` indices, instance threading via `requestedRef` (index into the requested pom's
declarations, not a value copy), event **multiset** (ordering excluded by design), exception
class + normalized message.

Normalization rules (`SnapshotNormalizer`): absolute paths → `<path>`; `localhost:<port>` →
`<local>`; dated-snapshot timestamps when generated live → `<ts>`; message = first line, URL
authorities normalized. Mask registry: `parity/masks.txt`, lines of `ledgerId jsonPathPrefix` —
a mask without a ledger id fails the build.

## 3. Corpus + record/replay + ground truth (slice B — developed in `maven-resolution-plan/phase0-tools/`, standalone Gradle project, final package `org.openrewrite.maven.parity.corpus`)

Standalone during development (avoids gradle project-lock contention with slice A; same proven
pattern as the spikes); merged into rewrite-maven test sources at review.

| File | Responsibility |
|---|---|
| `RecordingHttpSender.java` | Wraps any `HttpSender`. `RECORD`: pass through, persist `(meta.json, body.bin)` keyed `sha256(method + canonical-url)` under a store dir. `REPLAY`: serve from store; miss → hard failure (hermeticity is the point). Store layout: `store/<2-char-prefix>/<hash>/`. |
| `corpus.yaml` | Committed manifest. Entry: `{name, kind: pom|reactor, source: {gav: "g:a:v"} or {git: {url, tag}}, groundTruth: [dependency-tree, effective-pom], notes}`. Seed set: ~20 Central poms chosen for feature coverage (spring-boot-starter-web, quarkus-bom importer, guava, a `${revision}` reactor, a ranges user, a dated-snapshot consumer…), 2 real reactors (apache/maven@maven-3.9.16, apache/camel@<pinned> for the large-reactor benchmark), every fixture from slice A. |
| `CorpusFetch.java` (main) | Materializes entries into gitignored `.corpus/` (git clone --depth 1 for reactors; pom download via the recording sender for Central entries). |
| `GroundTruthCapture.java` (main) | Runs real Maven 3.9.16 (downloads the distro into `.corpus/tools/` if absent) per entry: `dependency:tree -Dverbose` and `help:effective-pom`, output into the store next to the recorded HTTP responses. These files are the FIX-vs-BUG arbiter for ledger triage. |
| `CorpusResolutionRunner.java` (main) | Phase-0 version: resolve every corpus entry with the legacy engine under REPLAY, snapshot, write `snapshots/<entry>.json`. Phase 2+ reuses it dual-engine. Twice-run determinism check included. |

Committed to git: manifest, slice-A fixture stores (small, synthetic). NOT committed: `.corpus/`
and real-world HTTP stores (CI cache keyed by manifest hash; refresh = deliberate re-record).

## 4. Synthetic behavior corpus (slice C — after A lands; `parity/synthetic/`)

MockWebServer-backed parameterized tests for what Central can't exercise: mirror matrix
(`*`, `external:*`, `!excl`), auth + anonymous-fallback, snapshot metadata + classifier
selection, dead repos (negative caching, unreachable-endpoint set), `file://` repos, HTML-index
metadata (pending the keep/kill call), pom-less jars (pin against Maven's missing-descriptor
tolerance). Each test asserts a snapshot, so in shadow mode these compare engines automatically.
HEAD responses must carry no body (spike lesson — poisons MockWebServer keep-alive).

## 5. Ledger — `doc/maven-resolution-ledger.md` (in-repo, reviewed like code)

One table, one row per divergence:

```
| id | area | current behavior | Maven behavior | class | evidence | pinning test | status |
```

- `class` ∈ `ALIGN_TO_MAVEN` (default for anything semantic) | `KEEP_REWRITE` (output
  shape/API/seams only — each row must justify itself against DESIGN.md §0).
- `evidence` = corpus entry / ground-truth file / spike reference; `pinning test` = the unit
  test asserting Maven-identical behavior (added with the fix; test method javadoc carries the
  ledger id).
- Seeded in Phase 0 from DESIGN.md §9 plus every determinism-audit finding and every mask.

## 6. Benchmarks

The comparative harness stays canonical at `maven-resolution-plan/spike/benchmark/` until the
shaded engine exists (single-JVM JMH comparison is blocked on shading; the spike's split-JVM
design is correct until then). Phase-0 additions: the apache/camel large-reactor entry (the
`ModelCache` clone-scaling question), and baseline results committed as
`maven-resolution-plan/benchmarks/baseline-<date>.json`. Graduation into
`rewrite-benchmarks/src/jmh/java/org/openrewrite/benchmarks/maven/` happens in Phase 1.

## 7. Exit criteria

1. `DeterminismTest` green over all slice-A fixtures (masks only with ledger ids).
2. `IdentityContractsTest` + `SerializedLstCompatibilityTest` green.
3. Corpus: fetch + record + ground-truth capture complete for the seed set; `CorpusResolutionRunner`
   green under REPLAY with network disabled (hermeticity proven), snapshots committed to the CI
   cache, determinism holds corpus-wide.
4. Ledger exists with §9 seeds + audit findings; masks file empty or fully ledgered.
5. Large-reactor benchmark baseline recorded.

## 8. Delegation plan

- **Slice A** → implementation agent, directly in rewrite-maven test sources (sole gradle user
  of the repo during its run). Review: force-rerun, snapshot-format fidelity check against §2
  (the lossiness review is the critical one), fixture coverage check.
- **Slice B** → implementation agent, standalone in `phase0-tools/` (no gradle contention).
  Review: hermeticity (REPLAY with sockets disabled), store determinism.
- **Slice C, D** → after A/B review, next round.
- Work remains uncommitted in the `maven-resolution` worktree until Jon calls for a branch +
  commit; nothing lands on `main` directly.
